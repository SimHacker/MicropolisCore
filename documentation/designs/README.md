# MicropolisCore Design Notes

This directory captures design rationale that should outlive chat context.

## Documentation layout

**`documentation/`** is the umbrella for manuals, talks, HTML bundles, **frozen snapshots** (for example **`documentation/openlaszlo/`** with `*.lzx` and scratchpads like **`TODO.txt`**), long-form **historical** archives, and this **`designs/`** subtree for **ongoing** intent: Wasm bridge, command path, multiplayer metaphors. Everything here is documentation; **`designs/`** is simply where we keep specs that track what we intend to build next.

When an older note foreshadows current work (city branching, social play, Git-as-multiverse), prefer a **link** from the archive into this folder instead of duplicating prose — see **`platform-lineage-index.md`**.

Open work for all of these areas is consolidated at **[documentation/TODO.md](../TODO.md)**.

- `characters-as-hydrogen.md` — **The top-level framing.** Characters are the *hydrogen* of the **Micropolis Federation** — the most abundant content-atom, the one with the highest valence-count, the one that binds with all the rest. Lots, objects, behaviors, appearances, memories, stories, and sounds are the other elements; households, neighborhoods, cities, towers, and dreams are molecules. Multi-resolution (same character at zone-aggregate / lot-render / Sim / MOOLLM-citizen / soul-file resolution) and multi-universal (same character across Micropolis Home, Micropolis City, the speculative Micropolis Tower, and Micropolis Dream — plus the wider [federation peer games](federation-peer-games.md)). Cross-license substrate: works with GPL3 (Micropolis itself) and proprietary games (The Sims) via documented file-format access. "Federation" deliberately, not "franchise" — Star-Trek-style cooperative association of sovereign open-source projects, not a commercial licensing scheme. *Read this first if you're trying to understand what the Federation actually is.*
- `simopolis.md` — **Micropolis + The Sims under one umbrella.** The unified Simopolis vision: two-resolution world (historically grounded in Will Wright's 1996 Stanford "Dollhouse" demo, which loaded a SimCity file and zoomed to street level), the Uplift character bridge, TypeScript-first IFF pipeline, ecosystem design philosophy. The primary strategic document for the Sims integration work in MicropolisCore.
- `moollm-microworld-os.md` — **The agent layer.** MOOLLM as the microworld operating system for Simopolis: filesystem-as-world, characters as directories, Mind Mirror + sims-traits double model, Speed-of-Light society-of-mind simulation, the Incarnation contract, the Bifrost bridge to The Sims. Deep-dive synthesis of the MOOLLM repo's relevant skills and designs.
- `the-tornado-and-the-archives.md` — **The recovery pipeline.** How a "tornado" sweeps the Internet Archive's preserved snapshots of dead and abandoned Sims sites (The Sims Exchange, preserved Yahoo Groups, Geocities-era story sites, defunct fan pages) and deposits the recovered families into Micropolis residential zones, with full provenance, takedown discipline, and living-person policy. Live commercial sites (SimSlice, SimFreaks, ZombieSims, TSR) are explicitly out of scope.
- `map-compositing-and-measurement.md` — **Software + WebGPU dual path**, composable overlay planes (MOP/color maps), JSON measure protocol (two-way bindable props), `CursorLayer` coordinator, client-GPU-upload economics, **§5** vote preview (dialog + in-world bounce; quorum height; WYSIWYG assent); **§3.5** north star: collaborative tools, tool-as-vehicle, Factorio-like blueprint params (design only).
- `the-computer-as-portal.md` — **Worlds embedded in worlds.** The Sims Computer object uplifted: custom IFF content (Uplifted Computer, CD-ROMs, Save-Game Disks, foreign Photo Albums) for the EA-published Sims 1 that "installs" Micropolis as a playable PC app, embeds Sims save files inside other Sims save files, and embeds Micropolis cities inside The Sims via screen-snapshot SPR2 sprites. MOOLLM URL scheme. The Adventure Compiler's first player-visible target.
- `the-imagine-loop.md` — **LLM-as-narrator, not LLM-as-simulator.** The architectural alternative to duplicating the Sims runtime. Examine a parsed save → Imagine outcomes via LLM (time skips, what-ifs, retroactive backstory, dreams, cheats with narrative) → Edit the high-level YAML representation → Inject a valid `.iff` back into the EA-published Sims 1, which plays it normally. The two-product naming (Micropolis City + Micropolis Home) is introduced here.
- `family-album-as-storymaker.md` — **Family Album as branching/merging community graph.** The Sims Family Album extended into a branching, merging, geo-tagged graph of scenes — "snippets of DNA" weavable between authors' stories. The 35-year lineage from SimCity (1989) → HyperLook SimCity (1991) → SimCityNet (1993) → DreamScape (1995) → The Sims Family Album (2000) → MediaGraph (2010) → Bar Karma / StoryMaker / Urban Safari (2011) → MicropolisCore now. Five navigation views (Map / Road / Pie-menu / Album / Branching-Story). Federated by git; round-trips to documented Sims 1 album book IFFs.
- `federation-peer-games.md` — **Bridge-target catalogue.** Which games make great spiritual + technical fits for Bifrost character bridges. Top tier: Crusader Kings III, RimWorld, Stardew Valley, Dwarf Fortress. Strong tier: Bannerlord, Bethesda games (Skyrim/Fallout/Starfield), Two Point sims, Cities: Skylines. Tabletop bridges: Foundry VTT, D&D Beyond, Pathbuilder. Plus lineage extensions (Spore, SimCity 4, SimEarth) and an explicit anti-target list (Sims 4, BioWare/EA games, live-service shooters, gacha). Onboarding playbook for adding new peers to the Federation.
- `sims-content-registry.md` — **Dependency resolution toolset.** SQLite-backed scan of user content directories; validation walk over save files; repair strategies for missing or colliding references; LLM-assisted matching when there's no exact name match. Cross-cuts Phase 0 uplift, the Tornado, the Adventure Compiler, and the Imagine Loop — every flow needs dependency-correct output.
- `tomodachi-life-and-simopolis.md` — **Cultural / strategic notes.** Comparison with Nintendo's *Tomodachi Life: Living the Dream* (April 2026). What Tomodachi does well, what Nintendo restricted (image sharing, online Mii sharing, concert hall — citing "spirit of the game" worries), the architectural difference between a Nintendo walled-garden console title and an open content companion to The Sims, and six concrete small Simopolis features inspired by the comparison (household-lingo file, voice-mode toggle, region-targeted starter packs, Reality-TV preset, Mii-News-style newscast preset, fukuwarai-style emergency face builder).
- `og-cozy-games.md` — **Historical receipts + cultural framing.** Why The Sims (2000), and SimCity / Micropolis before it, are the **OG cozy games** that the modern Stardew / Animal Crossing / Tomodachi LTD genre descends from. With **primary-source historical receipts**: Don's *Sims Design Document Draft 3* (8/7/1998) and *Draft 5* (8/31/1998) review documents archived at [donhopkins.com/home/TheSims/](https://donhopkins.com/home/TheSims/), where inclusive procedural-rhetoric design (same-sex relationships as a *rule of the simulated world*, not a representation overlay) was set in the Maxis design phase 14 years before Gamergate. Documents how The Sims's commercial success (best-selling PC game of all time, majority-female playerbase, billions in revenue) structurally undermines every load-bearing Gamergate claim — and why Gamergate quietly avoided ground zero. Names the cozy-games dismissal cycle (girly games → casual games → cozy slop, same audience, three labels, same misogyny) and the design discipline that follows from the documented record. With citations to Ian Bogost (procedural rhetoric), Izzzyzzz's May 2026 *Cozy Games and Misogyny* video, Alexander Avila's *Did The Sims make you gay?* essay, the Fortune 25th-anniversary Sims piece, and the journalistic / academic consensus on Gamergate → alt-right → MAGA (Axios, Britannica, WIRED, Poynter, University of Melbourne, ABC Australia).
- `designing-inward-miyamoto-principles.md` — **Craft / design-discipline notes.** Shigeru Miyamoto's design principles, drawn from Don's first-hand HN notes, Miyamoto's GDC 1999 and 2007 keynotes, Wikipedia, and *Building SimCity* (Gingold, 2024). Design inward from the player's hands and face; fukuwarai as the lineage of our Character Customization Studio; the Simulator Effect ("implication is richer than simulation"); *kyokan*; tools first, content second; subvert your own genre; **Scott McCloud's masking effect** (abstract character, detailed world — the design principle that explains The Sims 1's stylized Sims in photoreal houses, the success of original-Miis and Tomodachi LTD's toon-Miis, and the failure of Switch Sports's Sportsmates). With concrete features inspired (cheap interpretive metadata; empty-state defensiveness; spectator-mode for Family Album books; **and a major Twitch-friendly subsection** — thirteen concrete streaming-integration features from chat-as-writers'-room to an official Twitch Extension to a vision-LLM screen-scraping agent (modern *Simplifier*) for catalog narration / accessibility / online-library lookup, treating "watching the player IS the game" at Twitch-scale as a first-class supported workflow). Includes a clear separation between *Miyamoto-the-designer* (deep respect) and *Nintendo-the-platform-owner* (legitimate criticism of recent restrictions), with **the same separation extended to Will Wright vs current Electronic Arts**, including the September 2025 EA → PIF / Silver Lake / Affinity Partners (Jared Kushner) $55B leveraged buyout context.
- `simopolis-uplift-roadmap.md` — **The build plan.** Phased, shippable roadmap for Simopolis: Phase 0 (one save end-to-end) through Phase 5 (federated mirror + recurring tornado), now including Phase 1C (Uplifted Computer) and Phase 1D (Imagine Loop). Cross-cutting work, risk register, open questions, definition of done.
